gitflow ​

GitFlow 是一种 Git 工作流，这个工作流程围绕着 project 的发布(release)定义了一个严格的如何建立分支的模型。它是团队成员遵守的一种代码管理方案。这只是一种建议，在团队合作中，具体项目中要灵活应用，不用可守成规，觉得不合理的地方可以自行修正。

txt
// 分支划分
- production
- release
- develop
- feature
- hotfix
